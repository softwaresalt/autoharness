Implement the replay-safe pre-execution telemetry context lifecycle ratified by
docs/decisions/2026-07-13-telemetry-metrics-reporting-ownership.md. This task
creates the stable execution identity and record handoff contract; backlogit
hierarchical-size capture remains in 079.013-T, and Ship template/dogfood wiring
remains in 079.015-T.

<!-- BEGIN:acceptance-criteria -->
- Tests are written first for a new CLI/library operation such as `autoharness telemetry begin` that runs before implementation/tool work, creates or accepts a caller-stable non-empty `epoch_id`, rejects empty/whitespace-only IDs, and writes a workspace-contained context artifact under the configured telemetry area (`.autoharness/metrics/contexts/` by default or a `contexts/` directory colocated with a custom `TelemetryConfig.database_path`).
- The context artifact contains context schema version, stable `epoch_id`, task/backlog/feature/shipment correlation, workspace/session/agent/phase metadata, branch/commit when known, `captured_at`, `snapshot_boundary = pre_execution`, optional frozen `WorkSizingSnapshot`, source/ruleset/version metadata, and a canonical digest; tests assert it contains no raw tool output, prompt bodies, stderr bodies, credentials, or unredacted argv.
- `autoharness telemetry record` can consume a context reference or embedded frozen context, preserves the same `epoch_id`, merges close-time roll-up metrics into the `ExecutionEpoch`, and must not re-read backlogit or generate a fresh UUID for replayable task-close records.
- Task-close validation rejects missing, empty, or whitespace-only `epoch_id` when no pre-execution context supplies the stable ID; `ExecutionEpoch.from_mapping` and CLI error tests cover omitted ID, empty string, and whitespace-only string.
- `RecordSummary`/CLI JSON output returns the stable `epoch_id`, context reference when supplied, canonical payload digest, and an idempotency outcome such as `created`, `idempotent_replay`, `partial_repaired`, `conflict_rejected`, or `disabled`.
- Replay tests cover identical replay, conflicting replay, and partial fail-open sink writes after one sink accepted the first payload: matching digest retries repair only missing sinks, conflicting digest retries preserve the first accepted record and write nothing new.
- Telemetry import-boundary tests still pass: `src/autoharness/telemetry/*` does not import gates, install/tune, `verify_workspace`, `schema_contracts`, agent-engram, CozoDB, or backlogit internals.
<!-- END:acceptance-criteria -->
