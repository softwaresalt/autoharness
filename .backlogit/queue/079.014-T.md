Implement the pre-execution `autoharness telemetry begin` context artifact
contract ratified by
docs/decisions/2026-07-13-telemetry-metrics-reporting-ownership.md. This task is
limited to creating/validating the stable execution identity and safe context
artifact before work begins. Task-close context consumption, `RecordSummary`,
idempotency coordination, and partial sink repair belong to 079.016-T; backlogit
hierarchical-size capture remains in 079.013-T.

<!-- BEGIN:acceptance-criteria -->
- Tests are written first for a new CLI/library operation such as `autoharness telemetry begin` that runs before implementation/tool work and creates a UUIDv4 epoch ID or accepts only a caller-provided UUID parseable by the standard UUID parser.
- Accepted caller IDs are normalized to the canonical lowercase `uuid.UUID(value).hex` form already produced by the current `ExecutionEpoch` default; arbitrary non-empty strings, empty/whitespace-only values, path separators, traversal tokens, and absolute-path-looking values are rejected.
- The context directory is derived from `TelemetryConfig` and must resolve inside the workspace: `.autoharness/metrics/contexts/` by default, or `contexts/` colocated with a custom workspace-contained `database_path`. Tests reject out-of-root custom telemetry paths, symlink escapes, absolute context refs, traversal, and separator-bearing IDs.
- The context path is computed as `{context_dir}/{canonical_epoch_id}.json` from the canonical ID only. Tests prove the resolved path remains under the resolved context directory and workspace root before any write occurs.
- Context creation is atomic/exclusive: begin must not overwrite an existing context. Repeated or concurrent begin with the same canonical ID and identical canonical context returns an idempotent result; same ID with different canonical context returns a conflict and preserves the original file.
- The context artifact contains context schema version, canonical `epoch_id`, task/backlog/feature/shipment correlation, workspace/session/agent/phase metadata, branch/commit when known, `captured_at`, `snapshot_boundary = pre_execution`, optional frozen `WorkSizingSnapshot`, source/ruleset/version metadata, and a canonical context digest; tests assert it contains no raw tool output, prompt bodies, stderr bodies, credentials, or unredacted argv.
- The returned `context_ref` is a repo-local reference under the configured context directory and includes enough metadata for 079.016-T to validate identity and digest before task-close record consumption.
<!-- END:acceptance-criteria -->

<!-- BEGIN:implementation-notes -->
