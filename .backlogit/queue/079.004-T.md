title: Detecting expected-versus-actual tool usage gaps
updated_at: 2026-07-13T22:29:26.4862208-07:00
---

Implement the expected-vs-actual tool usage gap signal defined by the ratified contract. A gap is counted when policy expected an indexed/routed tool but the actual event used raw search/read, degraded fallback, or an unavailable/stale route.

<!-- BEGIN:acceptance-criteria -->
- Unit tests cover expected Engram route used, expected route missed and raw read used, stale/unavailable route fallback, and no-gap cases when no route was expected.
- Gap records include `expected_tool`, `expected_reason`, actual route kind, `fallback_reason`, `freshness_state`, and safe `evidence_path` or `artifact_refs`.
- Gap counts roll up to `ExecutionEpoch` fields and can be queried from persisted sink data.
- The implementation does not enforce routing policy itself; it only records and reports evidence.
<!-- END:acceptance-criteria -->
