Publish the mandatory `ExecutionEpoch` v1.1 contract as root and versioned JSON schema mirrors and register that schema with the schema-contract inventory. This task is distinct from the `ToolTelemetryEvent` schema contract.

<!-- BEGIN:acceptance-criteria -->
- Root schema and versioned mirror are present for `ExecutionEpoch` v1.1 and describe every ratified epoch field, including root correlation fields (`feature_id` included), token-economics fields, provenance maps, route/operation/outcome roll-ups, per-tool gap maps, and the nested `WorkSizingSnapshot` payload.
- Both the root and versioned `ExecutionEpoch` v1.1 schema mirrors constrain `schema_version` with JSON Schema `const: "1.1.0"`, not only a plain string type, so the v1.1 schema rejects records labeled as another version.
- Nonnegative numeric epoch fields use JSON Schema `minimum: 0`, including token counts, cached/cumulative/context tokens, byte counts, durations, and result/route/tool counts; intentionally signed values such as `exit_code` or signed derived metrics are excluded.
- `src/autoharness/schema_contracts.py` registers the `ExecutionEpoch` schema contract without causing telemetry modules to import `schema_contracts`.
- Tests validate root/versioned mirror parity and schema-contract registration.
- Schema descriptions make nullable estimates, zero counts, `metric_sources`, and `metric_quality` semantics explicit so legacy or unavailable metrics never imply precision.
- The `WorkSizingSnapshot` schema describes nullable task/feature/shipment planned size labels, `snapshot_at`, `snapshot_boundary` with the ratified `pre_execution` value, per-level source/ruleset metadata, feature child count/histogram/hash, shipment manifest count/histogram/hash, and explicitly forbids fake numeric point fields unless a future named/versioned mapping is added by a schema bump.
<!-- END:acceptance-criteria -->

<!-- BEGIN:implementation-notes -->
Honor the repository schema mirror rule: update both `schemas/<epoch-name>.schema.json` and `schemas/<epoch-name>/1.1.0.schema.json`, plus `src/autoharness/schema_contracts.py`.
