Build the autoharness-owned telemetry reader layer for local epoch sinks resolved through `TelemetryConfig` (default `.autoharness/metrics/` plus workspace-contained custom database paths and their colocated JSONL files). Keep this task scoped to reading both existing persistence formats and normalizing records into the ratified `ExecutionEpoch` v1.1 shape, including optional `WorkSizingSnapshot`; aggregation, derived metrics, slicing, rendering, and live backlogit hierarchy access are separate downstream tasks.

<!-- BEGIN:acceptance-criteria -->
- Reader tests resolve inputs through `TelemetryConfig` and cover both local SQLite and JSONL epoch sinks for the default `.autoharness/metrics/` location and a workspace-contained custom `database_path` whose JSONL path is colocated with that database; support for each existing sink is mandatory, not optional.
- Legacy v1.0 epoch records normalize to the v1.1 shape with every metric that cannot be established as observed marked `unavailable` or `not_applicable`, including pre-existing v1.0 economic fields whose numeric `0` values came from defaults rather than observation; normalized output is never a hybrid `schema_version: 1.0.0` record containing v1.1 fields, and copies existing `task_id` into `backlog_item_id` for task epochs when the new correlation field is absent.
- Missing, partial, disabled, or absent telemetry inputs degrade to explicit empty/unavailable reader results without throwing or inventing zero-valued metrics.
- Reader canonicalization deduplicates by `epoch_id` before returning records: within JSONL, the last valid occurrence for an epoch ID wins; across SQLite and JSONL, callers either select one configured source or combined-source mode applies fixed precedence of SQLite over JSONL for matching `epoch_id` values.
- Reader tests cover identical duplicate JSONL retry appends, conflicting duplicate JSONL records, and mirrored SQLite+JSONL records. Conflicting duplicates are surfaced diagnostically and every `epoch_id` appears at most once in reader output.
- Reader outputs expose persisted epoch records only, with missing work-sizing data represented as explicit unavailable/null snapshot fields; no time-series aggregation, derived efficiency formulas, report filtering, rendering, backlogit adapter, CLI command, or queryable event store is added in this task.
<!-- END:acceptance-criteria -->

<!-- BEGIN:implementation-notes -->
Primary files should be a telemetry reporting reader module plus focused reader/normalization tests. Autoharness owns this local epoch reader surface; backlogit may link to reports later but must not become the metric store. Do not require an external service, event table, or CLI command in this task.
