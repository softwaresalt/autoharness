Build the autoharness-owned telemetry reader layer for local epoch sinks under `.autoharness/metrics/`. Keep this task scoped to reading both existing persistence formats and normalizing records into the ratified `ExecutionEpoch` v1.1 shape; aggregation, derived metrics, slicing, and rendering are separate downstream tasks.

<!-- BEGIN:acceptance-criteria -->
- Reader tests cover both local SQLite and JSONL epoch sinks under `.autoharness/metrics/`; support for each existing sink is mandatory, not optional.
- Legacy v1.0 epoch records normalize to the v1.1 shape with additive fields and metric provenance marked `unavailable`, never a hybrid `schema_version: 1.0.0` record containing v1.1 fields.
- Missing, partial, disabled, or absent telemetry inputs degrade to explicit empty/unavailable reader results without throwing or inventing zero-valued metrics.
- Reader outputs expose persisted epoch records only; no time-series aggregation, derived efficiency formulas, report filtering, rendering, CLI command, or queryable event store is added in this task.
<!-- END:acceptance-criteria -->

<!-- BEGIN:implementation-notes -->
Primary files should be a telemetry reporting reader module plus focused reader/normalization tests. Autoharness owns this local epoch reader surface; backlogit may link to reports later but must not become the metric store. Do not require an external service, event table, or CLI command in this task.
