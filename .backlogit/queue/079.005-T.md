title: Building autoharness telemetry reporting library
updated_at: 2026-07-13T22:29:27.5228948-07:00

Add an autoharness-owned epoch-level reporting library that reads local telemetry sinks and produces deterministic time-series summaries for trends, regressions, cost/usage efficiency, offload, and expected-tool gap detection. Reports operate on persisted epoch fields, not a queryable event store or CLI command.

<!-- BEGIN:acceptance-criteria -->
- Reports can summarize a selected date/range or task/session/phase/backlog-item slice from both local SQLite and JSONL epoch data under `.autoharness/metrics/`; support for each existing sink is mandatory. Every slice key must be backed by a persisted `ExecutionEpoch` v1.1 field such as `session_id`, `task_id`, `backlog_item_id`, `phase`, `branch`, or `commit_sha`.
- Trend output includes token consumption, token generation, context-area estimates, COGS, duration, routed-vs-raw usage, avoided-read counts/token estimates, expected counts, observed counts, missing counts, and per-tool gap rates over time.
- Efficiency/cost-effectiveness metrics are deterministic: `net_offload_tokens = avoided_read_estimated_tokens - tool_output_estimated_tokens`, `consumption_generation_ratio = input_tokens / output_tokens`, `gap_rate`, and `cost_per_successful_epoch` or another documented denominator. A derived metric reports `unavailable` — never `0` and never an exception — when any operand is `unavailable`/null or its denominator is zero, such as zero successful epochs in the slice or `output_tokens == 0`.
- Output distinguishes observed, estimated, derived, and unavailable measurements.
- The reporting library degrades gracefully when telemetry is disabled, the DB is absent, or only partial v1.0 records exist.
- No CLI subcommand is added in shipment 092-S; CLI exposure is explicitly out of scope for this shipment.
<!-- END:acceptance-criteria -->

<!-- BEGIN:implementation-notes -->
Autoharness owns this local epoch time-series reporting surface. Backlogit may link to reports later but should not become the metric store. Do not require an external service, event table, or CLI command in this task.
<!-- END:implementation-notes -->
