<!-- BEGIN:description -->
Assess feasibility of WSL-backed local Linux build/test/lint/format gates that mirror GitHub Actions `ubuntu-latest` and feed the unified CI + local-gating primitive. Spike framing is updated in `docs/decisions/2026-07-13-wsl-local-linux-builds-spike.md`. **Operator Decision Required:** separately authorize read-only WSL/distro inspection, explicitly approve any destructive WSL installation/provisioning, choose target distro/toolchain parity, and decide an evidence publication/trust mechanism before any remote-CI reduction can be trusted.
<!-- END:description -->

<!-- BEGIN:dod -->
- Operator grants or denies read-only inspection permission for a selected machine to detect WSL/distro state.
- Operator separately approves any destructive/system-changing WSL installation, Windows feature enablement, distro setup, or package/toolchain provisioning under Principle VII before it occurs.
- Operator chooses a target distro/toolchain/parity policy and whether repos should be tested via `/mnt/c` paths or native WSL clones.
- Operator chooses an evidence publication/trust mechanism (for example signed commit status/check-run, attestation, committed metadata, uploaded artifact, or advisory-only). Remote CI remains authoritative until such a mechanism exists.
- A future spike defines and validates a Linux-build evidence record for the current commit before any hook/CI policy relies on it.
- BLOCKED-ON-OPERATOR because inspection/provisioning permissions and remote-CI trust policy require explicit operator decisions.
<!-- END:dod -->

<!-- BEGIN:goals -->
- Explore local Linux confidence without weakening the fail-closed GitHub Actions aggregation-gate model.
- Wire any approved WSL runner through the local-gating/pre-push primitive only after evidence freshness, parity, and publication/trust rules exist.
- Record local Linux-build evidence that can be cited in PR Local Review Readiness when valid, while recognizing that gitignored `.autoharness/gates/` evidence is not directly observable by GitHub Actions.
- Keep remote CI authoritative whenever WSL evidence is absent, stale, failed, unverifiable, unpublished/untrusted, or not accepted by branch rules.
<!-- END:goals -->
