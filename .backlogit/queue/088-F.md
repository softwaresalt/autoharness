Bounded, opt-in experiment/benchmark validating postToolUse-based tool-output compression with GitHub Copilot CLI as the PRIMARY host. The 2026-07-15 spike (stash AF767A44) CONFIRMED via GitHub Docs that Copilot CLI exposes a postToolUse hook whose modifiedResult replaces a tool result before it enters the model transcript — refuting 086-F's MCP-only assumption. This feature executes the experiment the spike designed: a throwaway, flag-gated prototype (compression hook + byte-equivalent retrieval MCP tool + containment-safe local store), an AUC token-savings measurement harness, a benchmark corpus over real autoharness commands, decline-case/negative-control validation, and an operator decision memo (accept/narrow-pilot/reject). Non-goals: NO default/production install, NO harness artifacts depending on compression, NO second graph stack (agent-engram remains graph authority), NO schema or CLI-distribution changes. Relates to 086-F (authoritative CCR store/retention/storage/benchmark-safety analysis, reused not duplicated).

<!-- BEGIN:dod -->
- Copilot CLI postToolUse interception surface is exercised by a working throwaway prototype behind a disabled-by-default flag.
- AUC token savings are measured on real autoharness commands over 1/3/5/10 turns under two tokenizers, net of placeholder overhead.
- Decline cases and negative controls pass: secrets/gate-verdicts/tiny-outputs/unwritable-store are declined; rejected attempts leave no durable store row; failure outputs are untouched.
- Byte-equivalent retrieval (full or tested pagination, no silent truncation) is proven before any positive savings result is reported.
- Store is containment-safe (Constitution IV) under .autoharness/cache/brainspace/ with TTL/size cap/purge/gitignore/staged-file guard.
- Ends with an operator decision memo: accept, narrow-pilot, or reject.
<!-- END:dod -->
