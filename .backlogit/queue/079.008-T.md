<!-- BEGIN:acceptance-criteria -->
- Root schema and versioned mirror are present for `ToolTelemetryEvent` v1.0 and describe identity, correlation, tool, timing, outcome, token economics, provenance maps, offload, retrieval health, evidence, and safety fields from the ratified decision.
- Both the root and versioned `ToolTelemetryEvent` v1.0 schema mirrors constrain `schema_version` with JSON Schema `const: "1.0.0"`, not only a plain string type, so the v1.0 schema rejects records labeled as another version.
- Nonnegative numeric event fields use JSON Schema `minimum: 0`, including token counts, cached/cumulative/context tokens, byte counts, durations, result/route/tool counts, and retry counts; intentionally signed values such as `exit_code` or signed derived metrics are excluded.
- The JSON Schema types `server_name` as nullable (`string` or `null`) because `cli`, `shell`, `builtin`, and direct `api` events may have no MCP server.
- The JSON Schema declares the fixed minimum `required` set ratified by the decision: `schema_version`, `event_id`, `timestamp`, `tool_surface`, `server_name`, `tool_name`, `operation`, `status`, `retry_count`, `degraded_mode`, `sensitivity`, `redaction_applied`, `metric_sources`, `metric_quality`, and `artifact_refs`; required nullable fields such as `server_name` are present with `null` when not applicable.
- Schema conditions and tests reject any populated metric field that lacks a same-named entry in both `metric_sources` and `metric_quality`; adapters may not emit schema-valid metrics without provenance.
- The JSON Schema encodes `route_kind` and `freshness_state` as `oneOf: [ {enum: [...well-known values...]}, {type: string, pattern: "^x-[a-z0-9-]+$"} ]` (plus `null` where optional) so pack adapters can emit namespaced extensions without being rejected by the forward contract.
- `src/autoharness/schema_contracts.py` registers the `ToolTelemetryEvent` schema contract without causing telemetry modules to import `schema_contracts`.
- Tests validate root/versioned mirror parity and schema-contract registration for the event contract.
