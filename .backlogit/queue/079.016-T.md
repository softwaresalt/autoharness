Implement task-close consumption of the pre-execution telemetry context from
079.014-T and coordinate replay-safe `autoharness telemetry record` behavior.
Sink-specific first-write persistence remains in 079.003-T, and reader
canonicalization remains in 079.005-T.

<!-- BEGIN:acceptance-criteria -->
- `autoharness telemetry record --context-ref` accepts only repo-local context references produced by `telemetry begin`; before reading, it rejects absolute paths, traversal, separator tricks outside the expected path shape, symlink escapes, and any resolved path outside the configured context directory/workspace root.
- Record tests validate context identity and integrity: the filename stem and stored context `epoch_id` must both equal the canonical UUID hex ID, the context digest must match the canonical context content, and a caller-provided epoch ID or embedded context that disagrees with the context reference is rejected.
- The record path merges the frozen identity/correlation/sizing context with close-time roll-up metrics, preserves the stable `epoch_id`, and must not re-read backlogit size, hierarchy, shipment membership, or other mutable planning state at close.
- Replayable task-close validation rejects missing, unparsable, empty, or whitespace-only `epoch_id` when no valid pre-execution context supplies the stable ID; it does not silently generate a fresh UUID for replay safety.
- `RecordSummary`/CLI JSON output returns stable `epoch_id`, `context_ref`, canonical context digest, distinct canonical epoch payload digest, and an idempotency outcome such as `created`, `idempotent_replay`, `partial_repaired`, `conflict_rejected`, or `disabled`.
- Idempotency coordinator tests cover identical replay, conflicting replay, and partial fail-open sink writes after one sink accepted the first payload: matching payload digest retries repair only missing sinks, while conflicting digest retries preserve the first accepted record and write nothing new.
- Tests explicitly distinguish context digest from epoch payload digest: a valid context digest does not allow replacing a conflicting epoch payload, and a matching payload digest does not allow reading an unsafe or mismatched context reference.
- Telemetry import-boundary tests still pass: `src/autoharness/telemetry/*` does not import gates, install/tune, `verify_workspace`, `schema_contracts`, agent-engram, CozoDB, or backlogit internals.
<!-- END:acceptance-criteria -->
