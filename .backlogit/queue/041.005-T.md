Create the release pull request, wait for CI to go green, apply the P-014 Copilot Review Merge Gate (§1.9), then merge with a merge commit (P-009 — squash and rebase are forbidden).

## Pre-conditions (all must be met before this task starts)

- ✅ `041.001-T` done: closure branch merged to `main` (PR #93, commit `e52e785`)
- `041.002-T` done: all four version surfaces at `1.4.4`
- `041.003-T` done: CHANGELOG `## 1.4.4` section present and non-empty
- `041.004-T` done: all parity checks pass

## Steps

### 1. Push release branch
```
git push origin chore/release-1.4.4
```

### 2. Create PR
```
gh pr create \
  --title "chore(release): autoharness v1.4.4" \
  --body "Release v1.4.4 — P-014 Copilot Review Merge Gate hardening and 036-S closure.\n\nChanges:\n- Bump version 1.4.3 → 1.4.4 across all version surfaces\n- CHANGELOG entry for v1.4.4\n\nScope (since v1.4.3):\n- PR #90: P-014 Copilot Review Merge Gate (§1.9 defense-in-depth + §1.10 closure polling)\n- PR #91 + closure (PR #93): 036-S backlog reconciliation and archive\n\nPart of shipment 037-S." \
  --base main \
  --head chore/release-1.4.4
```

### 3. Request Copilot review
Per §1.1 of github-pr-automation.instructions.md.

### 4. CI gate
Wait for GitHub Actions CI to go green. Do NOT proceed if any check is red.

### 5. P-014 Copilot Review Merge Gate (§1.9 — NON-NEGOTIABLE)
Run the full §1.9 readiness query (paginated GraphQL):
- Check 1: No pending Copilot review request
- Check 2: Copilot review covers current HEAD (commit OID matches headRefOid)
- Check 3: Zero unresolved Copilot-authored review threads

Address any review findings per §1.3–§1.7 fix cycle before the gate will pass.

**If any of the three checks fail: HALT. Do not merge. Report to operator.**

### 6. Merge with merge commit (P-009)
```
gh pr merge --merge --subject "chore(release): autoharness v1.4.4"
```
Never use `--squash` or `--rebase`.

### 7. Sync local main
```
git checkout main
git pull origin main
```
