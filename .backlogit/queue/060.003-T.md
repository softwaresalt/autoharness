Update Stage and Orchestrator guidance so pipelining no longer normalizes parallel implementation branches or worktrees. In templates/agents/.stage.agent.md.tmpl and .github/agents/.stage.agent.md, make the Stage exception explicit: Stage may use a separate worktree only for time-boxed spike/research investigation, must not implement or modify templates/source/config there, must record the spike context, and should clean up or hand off findings before Ship execution. In templates/agents/_orchestrator.agent.md.tmpl and .github/agents/_orchestrator.agent.md, replace the current pipelined-mode constraint that both agents are on different branches with P-016-compliant sequencing: no concurrent Stage/Ship implementation branches or parallel worktrees; queued planning may proceed only when it does not create a parallel implementation branch, with Stage spike/research worktrees as the narrow exception. Acceptance: Orchestrator no longer asks Stage and Ship to operate on different implementation branches; Stage role boundary states the research exception and its limits; mirrors match templates.
