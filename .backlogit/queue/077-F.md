Consolidate the workspace-root `.mcp.json` launcher strategy, which today mixes three styles: native binaries on PATH (`backlogit`, `engram`, `graphtor-docs`), plain `npx` (`context7`), and `pwsh -NoProfile -Command` wrappers (`tavily` guards a required env var; `github` reads the `gh` credential via command substitution and launches the DEPRECATED `@modelcontextprotocol/server-github`). The `pwsh` coupling hard-fails on POSIX dev boxes that do not ship PowerShell. Research is COMPLETE — see `docs/decisions/2026-07-12-mcp-launcher-consolidation-deliberation.md` (options A-E; recommended net direction: npx + native binaries + the official GitHub MCP server, consolidating away from pwsh). Implementation is OPERATOR-GATED and this feature is therefore blocked-on-operator.

<!-- BEGIN:dod -->
- Operator ratifies the `github` server target and the supported client set (the two open product/environment decisions).
- `tavily`'s `pwsh` guard is replaced by a standard per-server `env` field forwarding `TAVILY_API_KEY` via a non-literal reference (no `pwsh`/`sh`, no literal secret duplicated in config).
- `npx`-launched servers are pinned to exact versions; `gh` failure/empty-output is handled wherever a launcher still reads the credential.
- Automatable checks pass (JSON/schema validity, PATH resolution, direct MCP `initialize` handshake); an operator-present IDE/CLI restart confirms every server reconnects.
- `docs/environment-setup.md` is updated to match the chosen strategy and note any remaining external prerequisite.
- BLOCKED-ON-OPERATOR because the two ratification decisions are product/environment tradeoffs and implementation edits the LIVE, gitignored root `.mcp.json`, which can break MCP on the next IDE/CLI restart — unsafe to auto-apply while the operator is AFK.
<!-- END:dod -->

<!-- BEGIN:goals -->
- Retire the deprecated `@modelcontextprotocol/server-github` and adopt GitHub's official MCP server, routed by client/auth capability (hosted-remote, official local stdio binary, or Copilot-CLI built-in).
- Remove the `pwsh` OS coupling so `tavily`/`github` launch natively on Windows, macOS, and Linux.
- Preserve the no-token-duplication property (read the `gh` credential rather than copying a PAT into config), or accept a documented, operator-ratified tradeoff.
- Pin every `npx`-launched server (`tavily`, `context7`) to an exact reviewed version instead of `@latest`.
<!-- END:goals -->
