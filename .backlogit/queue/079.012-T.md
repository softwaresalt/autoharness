Build the report slicing and output layer on top of normalized readers and derived aggregation results. Keep this task scoped to selecting persisted correlation slices, presenting time-series summaries, and graceful operator-facing output behavior.

<!-- BEGIN:acceptance-criteria -->
- Reports can filter by persisted correlation fields only: `session_id`, `task_id`, `backlog_item_id`, `phase`, `branch`, `commit_sha`, and other ratified `ExecutionEpoch` v1.1 fields.
- Output includes token consumption, token generation, context-area estimates, COGS, duration, routed-vs-raw usage, avoided-read counts/token estimates, expected counts, observed counts, missing counts, per-tool gap rates, and derived efficiency metrics from 079.011-T.
- Output distinguishes observed, estimated, derived, unavailable, and not-applicable measurements without converting unavailable data into false precision.
- Report rendering degrades gracefully when telemetry is disabled, the DB is absent, JSONL is absent, or only partial v1.0 records exist.
