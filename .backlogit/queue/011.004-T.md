## Context

ATV's `pkg/plugingen/audit.go` scans harness artifacts for marketplace-incompatible content: hardcoded plugin cache paths, namespaced agent references, MCP plugin tool names. Uses regex rules with sortable findings. autoharness `verify_workspace.py` validates templates but has no equivalent portability assertion.

Goal: assess whether a portability audit assertion should be added to `verify_workspace.py` (or a new `PORTABILITY_ASSERTIONS` group) and what rules it would enforce. Deepens environment agnosticism (Core Rule 3).

## Acceptance Criteria

- [ ] Read `atv-starterkit/pkg/plugingen/audit.go` and `generate.go`
- [ ] Extract audit rule categories (path patterns, namespace patterns, tool name patterns)
- [ ] Map each category to potential autoharness violations (e.g., hardcoded `.github/local-agents/` paths, VS Code-specific references)
- [ ] Produce findings document at `docs/spikes/011.004-portability-audit-pattern.md` with:
  - ATV audit rule taxonomy
  - Candidate portability rules for autoharness (with regex examples)
  - Recommendation: add portability assertions to `verify_workspace.py` / add new assertion group / defer / reject
  - If adopt: proposed `verify_workspace.py` assertion structure and rule list
- [ ] Spike is done when the findings doc is committed; no `verify_workspace.py` edits in this task
