Add 4 role enforcement assertions to `src/autoharness/verify_workspace.py` and corresponding test cases.

**Design decision**: Implements Decision 4 from the deliberation — 4 verification assertions.

**Assertions to add**:
1. **Role Boundary table presence**: When both `stage.agent.md` and `ship.agent.md` exist in the target workspace, both must contain a `## Role Boundary (NON-NEGOTIABLE)` section.
2. **Role Boundary table consistency**: Stage's Forbidden column must include operations that Ship's Allowed column includes (and vice versa). No operation should be Allowed by both agents in the same category.
3. **Role enforcement instruction presence**: When two-agent model is detected (both agents present), `role-enforcement.instructions.md` must also be present in the target workspace instructions directory.
4. **No cross-contamination**: Stage agent body should not reference Ship-only operations (`claim_shipment`, `pr_create`, `git push`) outside the Forbidden column. Ship agent body should not reference Stage-only operations (`create_shipment`, `stash`) outside its allowed scope.

**Test cases** (in `tests/`):
- Pass case: both agents with consistent boundaries + instruction present
- Fail case: Ship missing Role Boundary table
- Fail case: role-enforcement instruction missing when both agents installed
- Fail case: overlapping Allowed operations between Stage and Ship
- Single-agent case: only one agent present — all 4 assertions skip gracefully

**Implementation notes**:
- Parse the Role Boundary table by finding the `## Role Boundary (NON-NEGOTIABLE)` heading and extracting the Markdown table rows
- Use simple text matching for cross-contamination (list of known operation keywords per role)
- Assertions should be conditional — they only run when both agents are detected

**Acceptance criteria**:
1. 4 new assertions in verify_workspace.py
2. All assertions are conditional on two-agent model detection
3. Test file with ≥5 test cases covering pass, fail, and skip scenarios
4. Tests pass when run with `pytest`
