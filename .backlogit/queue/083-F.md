<!-- BEGIN:description -->
Strengthen the harness's Engram-first structural-query routing so agents MUST route structural code questions — callers, callees, impact, symbols, inheritance/implementation relationships, and blast-radius checks — through the agent-engram tool family (`map_code`, `impact_analysis`, `list_symbols`, and related graph lookup tools) before grep or raw file reads. This is Option A from the TokenMasterX evaluation: build now, with no dependency on 079-F.

Scope is limited to tightening the existing `agent-engram` capability-pack overlay and retrieval-enforcement coordinator. It MUST NOT add a second graph stack, introduce TokenMasterX/graphify/codegraph as another graph authority, or duplicate graph ownership. Existing fallback discipline, direct-tool exemptions (literal/regex search, known exact path confirmation, trivial single-file lookups), freshness rules, and internal-first / no-public-web safeguards remain in force.
<!-- END:description -->

<!-- BEGIN:dod -->
- Structural code questions have explicit MUST-route-before-grep language in the agent-engram instruction and capability-pack enforcement coordinator.
- Precedence is clear: direct-tool exemptions first, otherwise agent-engram is the first route for structural code questions before raw grep/file reads.
- Existing fallback, freshness, and sensitivity safeguards are preserved; no public-web fallback is introduced for internal code or business-context queries.
- No second graph stack or graph-supplier abstraction is added; agent-engram remains the single graph authority.
- Verification and tune-drift checks assert the tightened structural-routing enforcement and maintain dogfood parity between templates and installed mirrors.
<!-- END:dod -->

<!-- BEGIN:goals -->
- Capture the near-term TokenMasterX token-economics win by making graph-backed structural navigation the default path through existing Engram tooling.
- Reduce repeated context spend from grep/raw file reads when a structural graph lookup is the correct first tool.
- Keep the retrieval-enforcement overlay small, auditable, and width-isolated so Ship can implement it without broad harness churn.
- Preserve the global-tool/local-output architecture and manifest checksum discipline for installed dogfood mirrors.
