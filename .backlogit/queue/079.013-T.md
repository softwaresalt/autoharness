Integrate the released backlogit hierarchical-sizing contract with the ratified
`WorkSizingSnapshot` telemetry payload as the capture producer used by
`autoharness telemetry begin`. The external prerequisite is now satisfied:
backlogit released the hierarchical-sizing contract (feature 108-F / shipment
099-S) with documented level-relative `size` semantics, stable CLI/read behavior,
stable hierarchy/shipment membership at the pre-execution boundary, and
re-estimation history/revision semantics that distinguish the planned estimate
captured before work ran from later changes. This task is therefore unblocked and
`queued`; source implementation may proceed under shipment 092-S.

Ratified reconciliation (task-only storage): the released contract stores `size`
as `custom_fields.size` on TASKS ONLY. Features and shipments never store a `size`
label; they expose a computed-on-read composition rollup: a `histogram` over
sized labels, a separate `unsized` count, de-duplicated tasks-only canonical
`members`, a `skipped` list of unresolved IDs, and a nullable `ruleset_version`.
This
task therefore snapshots the task-level size label directly and records
feature/shipment size labels as null-by-contract (not missing-data), alongside the
composition count/histogram/derived-hash for feature and shipment membership.

- Adapter/context tests prove the adapter captures once before execution and hands the exact frozen `WorkSizingSnapshot` to the 079.014-T telemetry context artifact without raw tool output or secrets.
- The adapter must not be called from task close: tests mutate backlogit size/membership after pre-execution capture and prove the already-created context still contains the originally captured snapshot. Task-close context consumption belongs to 079.016-T, and real Ship lifecycle end-to-end handoff belongs to 079.015-T.
- Immediately before the pre-execution composition/membership read, the adapter triggers a backlogit index sync/refresh so the captured `WorkSizingSnapshot` reflects current on-disk task/feature/shipment membership rather than a stale cached index; a test seeds a stale index, mutates on-disk membership out-of-band, and proves the pre-execution capture reflects the refreshed (synced) membership. This pre-capture freshness step is distinct from post-capture immutability: the sync happens once before the single capture to avoid freezing stale membership, after which the snapshot stays immutable and is never re-read or re-synced at close.
- Snapshot tests prove historical telemetry is immutable: later backlogit size, hierarchy, or shipment-membership edits do not rewrite already recorded epoch records.
- Feature and shipment composition tests derive count, histogram, and hash from the same canonical sorted unique task-ID set drawn from the rollup `members`. When membership is known, count equals `len(unique members)` and equals the sum of the `histogram` values plus the `unsized` count; `skipped` unresolved IDs are excluded from both the histogram and the unsized count and are surfaced diagnostically; duplicate IDs are collapsed, never counted twice. There is no `unavailable` histogram bucket.
- Membership hash tests use the ratified canonical digest: lowercase SHA-256 hex over a UTF-8 JSON array of unique ID strings sorted by ordinal/bytewise order and encoded without insignificant whitespace; unavailable membership data yields `null`, not a hash of an unknown set.
- The adapter treats `XS`/`S`/`M`/`L`/`XL` as ordinal level-relative labels only; it does not emit numeric point fields, convert labels to points, or report cost-per-size-point without a named/versioned mapping supplied by a future contract.
- Missing feature/shipment size labels, absent hierarchy data, or unavailable backlogit reads degrade to explicit null/unavailable snapshot fields without blocking telemetry emission.
- Backlogit remains the planned-sizing and hierarchy/membership source of truth; autoharness only records immutable epoch snapshots and reports over them.
