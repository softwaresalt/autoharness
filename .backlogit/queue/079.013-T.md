title: Snapshotting released backlogit hierarchical sizing into telemetry

Integrate the released backlogit hierarchical-sizing contract with the ratified
`WorkSizingSnapshot` telemetry payload. This task is deliberately blocked until
backlogit publishes a released contract/version that exposes optional `size`
frontmatter on task, feature, and shipment artifacts, with documented
level-relative semantics, stable CLI/read behavior for those fields, stable
hierarchy/shipment membership at the pre-execution boundary, and
re-estimation/history semantics that distinguish the planned estimate revision
captured before work ran from later changes.

<!-- BEGIN:acceptance-criteria -->
- This task remains `blocked` with label `blocked-on-external` until the released backlogit contract/version is available; no source implementation starts before that release exists.
- The released backlogit contract/version documents how a host retrieves the planned task/feature/shipment size labels before implementation/tool work begins, how it identifies the planned estimate revision when re-estimation is possible, and how it reads stable feature hierarchy plus shipment membership at that same pre-execution boundary.
- After the external release exists, tests snapshot task/feature/shipment planned size labels, `feature_id`, `snapshot_at`, `snapshot_boundary = pre_execution`, per-level `sizing_sources`, per-level planned-estimate `sizing_source_revisions`, per-level `sizing_ruleset_versions`, feature child task count/histogram/hash, and shipment manifest task count/histogram/hash into `ExecutionEpoch.WorkSizingSnapshot`.
- Snapshot tests prove the adapter captures once before execution and carries that already-captured snapshot into the epoch emitted at close; it does not re-read mutable backlogit state at close.
- Snapshot tests prove historical telemetry is immutable: later backlogit size, hierarchy, or shipment-membership edits do not rewrite already recorded epoch records.
- The adapter treats `XS`/`S`/`M`/`L`/`XL` as ordinal level-relative labels only; it does not emit numeric point fields, convert labels to points, or report cost-per-size-point without a named/versioned mapping supplied by a future contract.
- Missing feature/shipment size labels, absent hierarchy data, or unavailable backlogit reads degrade to explicit null/unavailable snapshot fields without blocking telemetry emission.
- Backlogit remains the planned-sizing and hierarchy/membership source of truth; autoharness only records immutable epoch snapshots and reports over them.
<!-- END:acceptance-criteria -->

<!-- BEGIN:implementation-notes -->
Primary implementation should be a small pre-execution backlogit sizing snapshot
adapter plus focused tests that feed the existing epoch model/sinks. If the
released backlogit API requires more than the task-granularity budget, split this
task before implementation. Remove `blocked-on-external` and move the task to
`queued` only after the released backlogit contract/version is available in a
backlogit-rooted session.
<!-- END:implementation-notes -->
