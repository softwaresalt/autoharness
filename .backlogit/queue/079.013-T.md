title: Snapshotting released backlogit hierarchical sizing into telemetry

Integrate the released backlogit hierarchical-sizing contract with the ratified
`WorkSizingSnapshot` telemetry payload as the capture producer used by
`autoharness telemetry begin`. This task is deliberately blocked until
backlogit publishes a released contract/version that exposes optional `size`
frontmatter on task, feature, and shipment artifacts, with documented
level-relative semantics, stable CLI/read behavior for those fields, stable
hierarchy/shipment membership at the pre-execution boundary, and
re-estimation/history semantics that distinguish the planned estimate revision
captured before work ran from later changes.

<!-- BEGIN:acceptance-criteria -->
- This task remains `blocked` with label `blocked-on-external` until the released backlogit contract/version is available; no source implementation starts before that release exists.
- The released backlogit contract/version documents how a host retrieves the planned task/feature/shipment size labels before implementation/tool work begins, how it identifies the planned estimate revision when re-estimation is possible, and how it reads stable feature hierarchy plus shipment membership at that same pre-execution boundary.
- After the external release exists, tests snapshot task/feature/shipment planned size labels, `feature_id`, `snapshot_at`, `snapshot_boundary = pre_execution`, per-level `sizing_sources`, per-level planned-estimate `sizing_source_revisions`, per-level `sizing_ruleset_versions`, feature child task count/histogram/hash, and shipment manifest task count/histogram/hash into the `WorkSizingSnapshot` object returned to the 079.014-T pre-execution context path.
- Adapter/context tests prove the adapter captures once before execution and hands the exact frozen `WorkSizingSnapshot` to the 079.014-T telemetry context artifact without raw tool output or secrets.
- The adapter must not be called from task close: tests mutate backlogit size/membership after pre-execution capture and prove the already-created context still contains the originally captured snapshot. End-to-end close/persistence handoff belongs to 079.015-T.
- Snapshot tests prove historical telemetry is immutable: later backlogit size, hierarchy, or shipment-membership edits do not rewrite already recorded epoch records.
- Feature and shipment composition tests derive count, histogram, and hash from the same canonical sorted unique task-ID set. When membership is known, count equals `len(unique_ids)` and equals the sum of histogram buckets including `unavailable`; duplicate IDs are collapsed and surfaced diagnostically, never counted twice.
- Membership hash tests use the ratified canonical digest: lowercase SHA-256 hex over a UTF-8 JSON array of unique ID strings sorted by ordinal/bytewise order and encoded without insignificant whitespace; unavailable membership data yields `null`, not a hash of an unknown set.
- The adapter treats `XS`/`S`/`M`/`L`/`XL` as ordinal level-relative labels only; it does not emit numeric point fields, convert labels to points, or report cost-per-size-point without a named/versioned mapping supplied by a future contract.
- Missing feature/shipment size labels, absent hierarchy data, or unavailable backlogit reads degrade to explicit null/unavailable snapshot fields without blocking telemetry emission.
- Backlogit remains the planned-sizing and hierarchy/membership source of truth; autoharness only records immutable epoch snapshots and reports over them.
<!-- END:acceptance-criteria -->

<!-- BEGIN:implementation-notes -->
Primary implementation should be a small pre-execution backlogit sizing snapshot
adapter plus focused tests over the 079.014-T `telemetry begin` context producer.
`autoharness telemetry begin` captures the snapshot after planning/manifest
assembly and before implementation/tool work, then stores it in the context
artifact. The adapter's integration surface is the begin-time host/runtime code
that prepares that frozen context; it must not live inside the sinks and must not
perform close-time backlogit reads. If the released backlogit API requires more
than the task-granularity budget, split this task before implementation. Remove
`blocked-on-external` and move the task to `queued` only after the released
backlogit contract/version is available in a backlogit-rooted session.
