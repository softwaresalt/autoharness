Integrate the released backlogit hierarchical-sizing contract with the ratified
`WorkSizingSnapshot` telemetry payload. This task is deliberately blocked until
backlogit publishes a released contract/version that exposes optional `size`
frontmatter on task, feature, and shipment artifacts, with documented
level-relative semantics and stable CLI/read behavior for those fields plus
hierarchy and shipment membership.

<!-- BEGIN:acceptance-criteria -->
- This task remains `blocked` with label `blocked-on-external` until the released backlogit contract/version is available; no source implementation starts before that release exists.
- After the external release exists, tests snapshot task/feature/shipment planned size labels, `feature_id`, `snapshot_at`, `snapshot_boundary`, per-level `sizing_sources`, per-level `sizing_ruleset_versions`, feature child task count/histogram/hash, and shipment manifest task count/histogram/hash into `ExecutionEpoch.WorkSizingSnapshot`.
- Snapshot tests prove historical telemetry is immutable: later backlogit size, hierarchy, or shipment-membership edits do not rewrite already recorded epoch records.
- The adapter treats `XS`/`S`/`M`/`L`/`XL` as ordinal level-relative labels only; it does not emit numeric point fields, convert labels to points, or report cost-per-size-point without a named/versioned mapping supplied by a future contract.
- Missing feature/shipment size labels, absent hierarchy data, or unavailable backlogit reads degrade to explicit null/unavailable snapshot fields without blocking telemetry emission.
- Backlogit remains the planned-sizing and hierarchy/membership source of truth; autoharness only records immutable epoch snapshots and reports over them.
<!-- END:acceptance-criteria -->

<!-- BEGIN:implementation-notes -->
Primary implementation should be a small backlogit sizing snapshot adapter plus
focused tests that feed the existing epoch model/sinks. If the released backlogit
API requires more than the task-granularity budget, split this task before
implementation. Remove `blocked-on-external` and move the task to `queued` only
after the released backlogit contract/version is available in a backlogit-rooted
session.
<!-- END:implementation-notes -->
