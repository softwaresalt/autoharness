<!-- BEGIN:description -->
Give auto-mergeinstall (install) and auto-tune (drift) the capability to maintain a dedicated GitHub Copilot code-review focus instructions file at `.github/instructions/copilot-code-review.instructions.md` — a path-specific instructions file that GitHub Copilot code review reads from the PR base branch. Frontmatter uses `applyTo: '**'` and `excludeAgent: 'cloud-agent'` so the guidance applies to GitHub Copilot's code-review bot but is skipped by the cloud coding agent, avoiding conflict with the repo-wide TDD/policy mandate in `.github/copilot-instructions.md`. Install generates the file from the workspace profile (backup-before-overwrite, never blind-clobber operator content); tune detects drift/staleness; verify checks coherence. AUTHORITATIVE (verified against docs.github.com "add-repository-instructions"): `excludeAgent` accepts exactly two values — `code-review` or `cloud-agent`; `cloud-agent` is the token that excludes the autonomous coding agent, which is what we want. `coding-agent` is NOT a valid value — do not substitute it.

DESIGN DECISION (adversarial plan review `plan-review-074`): deviate from the original stash text's literal `.github/copilot-instructions.md` target to a DEDICATED path-specific instructions file. Rationale: GitHub code review reads BOTH repo-wide and path-specific `*.instructions.md`; review-only de-prioritize directives placed in the repo-wide file would also reach the cloud coding agent and conflict with its TDD/policy mandate. `excludeAgent: 'cloud-agent'` isolates review-only guidance. Research-confirmed: `.github/copilot-review-instructions.md` is NOT a GitHub-honored path (internal-persona use only); GitHub honors `excludeAgent`; best-practice limit ~1000 lines with no unsupported instruction types.
<!-- END:description -->
