## Context

ATV's `pkg/sdk/intelligence.go` implements: rate limiting (min query interval, hourly budget), circuit breaker (3 consecutive failures → 5-min cooldown), and graceful offline degradation. autoharness has `circuit-breaker.instructions.md.tmpl` but without specific numeric defaults or state-tracking patterns.

Goal: assess whether ATV's concrete defaults and patterns should be back-ported to autoharness's circuit-breaker and concurrency templates. Deepens Primitive 5 (Guardrails).

## Acceptance Criteria

- [ ] Read `atv-starterkit/pkg/sdk/intelligence.go` and `intelligence_test.go`
- [ ] Extract specific defaults (min interval, hourly budget, failure threshold, cooldown duration)
- [ ] Compare against autoharness `templates/instructions/circuit-breaker.instructions.md.tmpl` and `templates/instructions/concurrency.instructions.md.tmpl`
- [ ] Produce findings document at `docs/spikes/011.003-sdk-guardrail-patterns.md` with:
  - ATV pattern summary (rate limiting, circuit breaker, offline degradation)
  - Gap analysis vs. current autoharness guardrail templates
  - Recommendation: adopt specific defaults / add state-tracking patterns / defer / reject
  - If adopt: proposed template delta (which variables or prose to add/update)
- [ ] Spike is done when the findings doc is committed; no template edits in this task
