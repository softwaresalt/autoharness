Add Auto-MergeInstall and Auto-Tune as elective subagents invocable by the Orchestrator. Update orchestrator.agent.md.tmpl to declare these as optional routing targets alongside Stage and Ship. Elective agents are invoked on operator request — not part of the default Stage→Ship pipeline. The Orchestrator routes to them the same way it routes to Stage or Ship (as subagents), with the constraint that elective agents must not run concurrently with active Ship work since they modify harness artifacts.

Key changes to orchestrator.agent.md.tmpl:
1. New "Elective Agents" section documenting Auto-MergeInstall and Auto-Tune, their purpose, invocation model, and how they differ from pipeline agents
2. New trigger phrases for elective invocation (e.g., "install harness", "tune harness")
3. New routing step in Required Steps for elective agent dispatch
4. Concurrency constraint: elective agents blocked during active Ship work
5. Subagent depth documentation for elective agents (same 3-hop model)
6. Model Routing table updated with elective agent entries
