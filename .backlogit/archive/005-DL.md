# Expand 034-S from Engram-only to full three-pack capability weave (agent-engram + agent-intercom + graphtor-docs)

## Problem Frame

<!-- BEGIN:problem-frame -->
Queued shipment 034-S is scoped to agent-engram integration only. The operator has directed that the next release unit should install and fully weave all three capability packs — agent-engram, agent-intercom, and graphtor-docs — so the dogfood workspace has all three packs coherently integrated across foundation docs, pipeline agents, and install/tune infrastructure. Additionally, execution must not begin on feat/pypi-first-distribution (033-S branch); it must start fresh from main after 033-S merges and v1.4.2 tag is pushed.
<!-- END:problem-frame -->

## Options

<!-- BEGIN:options -->
**Option A — Replace 034-S entirely**: Archive 034-S and all 036-x artifacts; create a new shipment 035+N covering all three packs from scratch.
- Pro: Clean slate, unified numbering.
- Con: Orphans six well-specified Engram tasks that are correct and complete; wastes planning work already done; introduces unnecessary re-specification risk.

**Option B — Amend 034-S additively**: Keep 036-F and its six tasks intact (Engram scope is still correct). Create two new feature artifacts — one for agent-intercom, one for graphtor-docs — each with their own width-isolated tasks. Update 034-S title, description, and branch-policy constraint to reflect the full three-pack scope.
- Pro: Zero rework on valid planning; additive change; each pack stays width-isolated per 2-hour rule; delivery ordering is explicit (Engram tasks can be sequenced before intercom and graphtor-docs).
- Con: Shipment manifest grows wider; requires careful branch-policy annotation to prevent Ship from starting on wrong branch.

**Option C — Create a parallel covering feature**: Keep 034-S but wrap everything under a new umbrella feature that delegates to three sub-features. Backlogit hierarchy supports this.
- Pro: Clean parent-child structure.
- Con: Adds a navigation layer without adding information; backlogit already shows shipment membership; unnecessary complexity.
<!-- END:options -->

## Chosen Direction

<!-- BEGIN:chosen-direction -->
**Option B — Amend 034-S additively.** 

Rationale:
1. The existing Engram tasks (036-F + 036.001-T through 036.006-T) are correct, well-specified, and unstarted — preserving them eliminates re-specification risk.
2. 034-S is still queued with no work started, so expansion is semantically clean.
3. agent-intercom and graphtor-docs are independent workstreams — separate features respect width isolation and the 2-hour rule without requiring a new umbrella layer.
4. A branch policy constraint note is added explicitly to the shipment and each new feature to make the start-from-main requirement agent-legible.
5. New artifacts: 037-F (agent-intercom full weave, 4 tasks), 038-F (graphtor-docs full weave, 4 tasks). Both added to 034-S manifest alongside existing 036-F.

Branch constraint (explicit, non-negotiable): ALL implementation for 034-S must begin from main AFTER 033-S (PR #85) merges and v1.4.2 tag is pushed. Branch: feat/capability-packs-full-weave (suggested). feat/pypi-first-distribution is 033-S scope — touching it for 034-S work is a P-001 violation.
<!-- END:chosen-direction -->
