Update the Search Protocol table and add a new Observability & Diagnostics Protocol section in templates/instructions/agent-engram.instructions.md.tmpl.

**Search Protocol table — add entry:**
- query_graph_neighborhood: "Traverse typed edges from a specific node to explore its local graph neighborhood" (this is the 033-S structured graph API, distinct from the existing query_graph for ad-hoc read-only queries)

**New section: Observability & Diagnostics Protocol**
Document 7 tools that provide workspace health, performance, and evaluation metrics:
- get_workspace_statistics — workspace-level indexing and coverage stats
- get_health_report — daemon and workspace health diagnostics
- get_evaluation_report — evaluation/quality metrics
- get_branch_metrics — per-branch indexing and activity metrics
- get_token_savings_report — token efficiency and savings data
- get_mutable_script_retry_metrics — retry/resilience metrics for mutable scripts
- index_git_history — git blame and history indexing for attribution

**Rules to include:**
1. Use diagnostics tools to verify workspace health before attributing search failures to query issues.
2. Prefer get_health_report for daemon troubleshooting; prefer get_workspace_statistics for indexing coverage gaps.
3. index_git_history is expensive — run it once per session when git attribution is needed, not on every query.

**Pattern:** Follow existing section style — table for tool routing, numbered rules for workflow guidance.
