## What

graphtor-docs is entirely absent from workspace metadata. Before the instruction template can be rendered, the two template variables must be resolved via workspace discovery. This is the discovery + metadata bootstrap task.

### Discovery steps

1. Locate the graphtor-docs binary (check PATH, `D:/Tools/`, `~/.local/bin/`, `.graphtor/`):
   - Run `where graphtor` or `Get-Command graphtor` (PowerShell)
   - Check `D:/Tools/graphtor.exe`
   - Record result as `GRAPHTOR_BINARY_PATH` (or `null` if not found — pack is still installable, binary path is informational)

2. Locate the sources configuration file:
   - Check `.graphtor/sources.yaml`, `.graphtor/config.toml`, `.graphtor/sources.toml`
   - Record result as `GRAPHTOR_SOURCES_PATH`

3. Check for MCP registration in `.mcp.json` (root) for a `graphtor` or `graphtor-docs` server entry

### Metadata updates

#### `.autoharness/workspace-profile.yaml`
Add a `graphtor_docs` section:
```yaml
graphtor_docs:
  detected: <true if binary found OR mcp_configured>
  mcp_configured: <true if .mcp.json has graphtor-docs server entry>
  config_paths: <list of mcp config files referencing graphtor-docs>
  sources_path: "<GRAPHTOR_SOURCES_PATH>"
  binary_path: "<GRAPHTOR_BINARY_PATH>"
  recommended: true
```
Add `graphtor-docs` to `harness_recommendations.capability_packs[]`

#### `.autoharness/config.yaml`
Add `- graphtor-docs` to `capability_packs:` list (after `agent-intercom`)

#### `.autoharness/harness-manifest.yaml`
Add `- graphtor-docs` to `capability_packs:` list
Add a `capability_pack_overlays` entry for `graphtor-docs` with overlay_targets and verification_checks stub (full contract added by GD-D)

## Acceptance Criteria

- workspace-profile.yaml has `graphtor_docs` section with `detected`, `mcp_configured`, `sources_path`, `binary_path`, `recommended` set correctly (binary_path/sources_path may be null if not found — null is valid, not an error)
- `config.yaml` `capability_packs` includes `graphtor-docs`
- `harness-manifest.yaml` `capability_packs` includes `graphtor-docs` with overlay entry
- Both template variable values (`GRAPHTOR_SOURCES_PATH`, `GRAPHTOR_BINARY_PATH`) are recorded somewhere agent-legible (workspace-profile.yaml preferred) before GD-B begins

## Constraint

Three related metadata files + discovery commands. ~30–45 min. No code changes.

## Dependency
