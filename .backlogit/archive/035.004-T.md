The working-tree version of docs/reference-library.md was rewritten for the (rejected) submodule-removal model and contains two passages that directly contradict the operator's decision to keep submodules. Simultaneously, the working-tree test `test_reference_library_is_indexed_without_git_submodules` asserts `.gitmodules` does NOT exist — which is wrong under the chosen direction.

## docs/reference-library.md changes

Remove or replace these two contrary passages:
1. "They are kept out of the autoharness Git install path so Windows `uv tool install` / `uv tool upgrade` runs do not fail while recursively checking out deep external trees into the `uv` cache." → Replace with accurate framing: submodules are registered in .gitmodules and can be initialised manually by maintainers; PyPI is the distribution path so end-users never traverse them.
2. "Do not vendor large upstream repositories into `references/` via submodules or copied trees — they break Windows `uv` Git installs by forcing deep path checkouts inside the `uv` cache." → Replace with: the references/ submodules are intentionally in-repo developer references; they are excluded from the PyPI wheel (not listed in pyproject.toml distribution targets); end-users install from PyPI and never encounter the submodule trees.

Add (or restore) a section explaining how maintainers initialise a specific submodule locally: `git submodule update --init references/<name>`.

## tests/test_verify_workspace.py change

Replace `test_reference_library_is_indexed_without_git_submodules` with `test_reference_library_submodules_are_registered` which asserts:
- `.gitmodules` EXISTS at repo root
- `.gitmodules` contains registrations for all six submodule paths: references/awesome-copilot, references/awesome-agent-skills, references/awesome-claude-skills, references/ai-skills, references/awesome-agents, references/agent-skills
- `docs/reference-library.md` does NOT contain 'kept out of the autoharness Git install path' (the contradictory passage)
- `docs/reference-library.md` DOES contain 'references/' (still documents the reference library)

Verify: `python -m pytest tests/test_verify_workspace.py::VerifyWorkspaceTests::test_reference_library_submodules_are_registered -v` passes.
