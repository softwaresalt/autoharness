## What

Ensure `install-harness SKILL.md` Step 1.3b contains a fully specified, first-class agent-intercom overlay contract alongside the agent-engram contract (added by WS-F in 036-F). Both must be present before 034-S is considered complete.

### Required contract elements (per harness-architecture.instructions.md)

1. **Eligibility signals**: `.mcp.json`, `.vscode/mcp.json`, `.cursor/mcp.json` references `intercom` server; `AGENTS.md` or agent files contain intercom tool names (heartbeat, broadcast, auto-check, transmit, standby)

2. **Recommendation logic**: Recommend when `mcp_configured: true` OR `detected: true` in `agent_intercom` profile section

3. **Overlay targets**:
   - `agent-intercom.instructions.md` (primary instruction file)
   - `pipeline-agents` (stage.agent.md, ship.agent.md — startup heartbeat, phase broadcasts, approval routing)
   - `destructive-action-workflows` (anywhere the harness gates destructive operations)
   - `operator-choice-surfaces` (stash triage, plan review, shipment assembly broadcasts)

4. **Behavior deltas**:
   - Session start: heartbeat/ping with status message
   - Before operator-facing choices: self-contained broadcast with IDs, priorities, and ordering rationale
   - Before destructive ops: auto-check → request clearance if not auto-approved
   - Phase transitions: broadcast at planning, build, review, verification, closure milestones
   - Fallback: `INTERCOM_DEGRADED` (non-destructive work continues; approval-dependent ops blocked)

5. **Verification checks**:
   - `agent-intercom.instructions.md` installed at `.github/instructions/`
   - stage.agent.md and ship.agent.md contain `heartbeat` / `INTERCOM_DEGRADED` references
   - No unresolved `{{VARIABLE}}` in agent-intercom.instructions.md (template has none)

6. **Tuning drift checks**:
   - `agent-intercom.instructions.md` checksum vs. template checksum
   - stage/ship agent intercom sections present (text search for `INTERCOM_DEGRADED`)
   - MCP config path in manifest matches `.mcp.json`

### Combined backlogit+intercom rule

Document that when both `agent-intercom` and `backlogit` packs are active, operator-facing backlog presentation must be self-contained broadcast: include item ID, priority, type, one-line summary, ordering rationale, and an explicit wait-for-confirmation statement. This rule is already in the installed instruction file — the SKILL.md must document when and how it is applied during installation.
