Add a gate between Step 1 and Step 2 in orchestrator.agent.md.tmpl: after Stage completes and produces a shipment_id, the orchestrator must verify that all staging artifacts (backlog items in .backlogit/) are committed and merged to the default branch before invoking Ship. Ship's P-011 Branch Creation Gate already requires a clean main, but it does not check whether the shipment manifest it will claim actually exists on main. The new gate: (1) Check git status for uncommitted backlog files, (2) If uncommitted, commit to a staging branch (chore/stage-{shipment_id}), create a PR, and merge, (3) Only after the staging PR is merged, proceed to invoke Ship. Add conditional logic: if branch protection is not detected (no PR required), Stage may commit directly to main. Consumes stash 47C56F4D.
