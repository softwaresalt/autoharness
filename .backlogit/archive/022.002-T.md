Create a new instruction template at `templates/instructions/context-efficiency.instructions.md.tmpl` (Primitive 1) that teaches agents context window hygiene across three layers:

**Layer 1 — Tool Result Offloading (Disk I/O)**:
- When a tool response exceeds ~2,000 tokens, write it to `{{DOCS_ROOT}}/scratch/{timestamp}-{summary}.md` and keep only a file path + 10-line preview in working context
- Prefer `view_range` over full file reads
- Prefer targeted grep over broad searches

**Layer 2 — Committed Change Eviction**:
- After committing and pushing, treat the diff as resolved
- Do not re-read committed file content unless the task requires it
- Reference the commit SHA, not the full diff

**Layer 3 — Proactive Compaction Trigger**:
- When the session has produced >10 checkpoint files or >40 memory files, invoke the compact-context skill before proceeding
- Do not wait for the environment to manage this — the agent must invoke compaction explicitly

- Valid YAML frontmatter with `description` and `applyTo: '**'`
- Markdown heading hierarchy per P-008
- Environment-agnostic — document what the environment handles vs. what agents must do manually
