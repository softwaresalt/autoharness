Create a new instruction template at `templates/instructions/role-enforcement.instructions.md.tmpl` that defines the pre-mutation self-check protocol for role enforcement.

**Design decision**: This implements Decision 1 (Option C) and Decision 2 (Option A) from the deliberation. The instruction teaches agents to self-check their Role Boundary table before any mutation.

**Template structure**:
1. YAML frontmatter with `description` and `applyTo: '**'` (applies to all files)
2. `# Role Enforcement` heading
3. Section explaining the self-check protocol:
   - At session start, read your own `## Role Boundary (NON-NEGOTIABLE)` section
   - Before any tool call that mutates state, classify the operation by category
   - If the operation appears in your Forbidden column, halt immediately and log P-010
   - If uncertain, default to forbidden (fail-safe)
4. Section listing the operation categories and examples of mutations per category
5. Section explaining P-010 violation recording and halt behavior
6. Section noting this instruction only applies when both `stage.agent.md` and `ship.agent.md` are present in the workspace (two-agent model)

**Reference**: Follow the format of existing instruction templates in `templates/instructions/` — especially `strict-safety.instructions.md.tmpl` which is another behavioral enforcement instruction.

**Template variables**: Use `{{PROJECT_NAME}}` where the project name is referenced. No other variables expected.

1. File exists at `templates/instructions/role-enforcement.instructions.md.tmpl`
