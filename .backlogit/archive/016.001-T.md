Create schemas/community-template-registry.schema.json defining the structure of the community template registry. Required fields per entry: template_id (unique slug), artifact_type (enum: agent, skill, instruction, prompt), title, description (what the template does and when to use it), source_repo (which references/ submodule it came from), source_path (original file path within the source repo), license (source repo license), template_path (path to the .tmpl file in templates/community/), applicable_profiles (array of technology profile tags like 'rust', 'go', 'python', 'typescript', 'any'), prerequisite_packs (array of capability pack names required for this template to be useful), tags (freeform string array for categorization), variables_introduced (array of new {{VARIABLE}} names this template uses, if any), primitives_deepened (array of primitive numbers this template relates to). Top-level structure: schema_version, templates[] array. Follow the same JSON Schema draft-07 pattern as backlog-tool-registry.schema.json and harness-config.schema.json.
