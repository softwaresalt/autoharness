title: Add distribution property to workspace-profile schema
updated_at: 2026-04-24T15:39:24.5687786-07:00
---

Add a `distribution` object to schemas/workspace-profile.schema.json with three fields: (1) `is_global_tool` (boolean) — true when the workspace is a globally-distributed tool that ships agents in its package, (2) `global_agents_dir` (string, nullable) — the directory where globally-distributed agents live (default `.github/agents`), (3) `local_agents_dir` (string, nullable) — override directory for local-only workflow agents that must not leak into the distribution package (default `.github/local-agents/`). Place the new property alongside the existing `vscode` and `tools` properties. Acceptance: schema validates, new property is optional, defaults documented.
