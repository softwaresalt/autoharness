Structural hardening of the policy registry, installed instructions, and agent/skill templates so that every PR merge — including post-merge closure PRs — requires both (a) zero unresolved Copilot review threads (§1.9 gate) and (b) explicit operator approval before merge is executed. Reactive remediation was required after PR #87, #88, and #89 because the merge gate was not enforced at the policy level and the critical §1.9 instruction file was not installed.

## Root Cause Summary

Three compounding gaps allowed PRs to merge with unresolved Copilot review comments:

1. `.github/instructions/github-pr-automation.instructions.md` is **not installed** in this workspace. The `pr-lifecycle` skill references it for §1.9 pre-merge gate, but the file does not exist — the gate protocol is unreachable at runtime.
2. No first-class policy (P-014) exists in the policy registry making the merge gate a named, telemetry-tracked constraint. The gate lives only in template text, not in policy enforcement.
3. The installed `.github/agents/ship.agent.md` Step 4 (PR Lifecycle) does not reference `github-pr-automation.instructions.md` or §1.9 at the agent level — it delegates entirely to the pr-lifecycle skill, which cannot load the gate protocol from a file that is not there.
4. Post-merge closure PRs have no Copilot poll requirement — the gate only covers the initial feature/chore PR.

## Acceptance Criteria

1. `github-pr-automation.instructions.md` is installed and extends §1.9 scope to post-merge closure PRs.
2. P-014 (Copilot Review Merge Gate) added to `workflow-policies.md.tmpl` with P-005 telemetry requirement.
3. Installed `.github/agents/ship.agent.md` Step 4 references P-014 and §1.9 explicitly; Step 5 adds Copilot poll gate for post-merge closure PR.
4. Template `ship.agent.md.tmpl` receives same hardening (future installs get it).
5. `pr-lifecycle/SKILL.md.tmpl` Step 5 adds last-mile §1.9 re-check after operator approval and before merge execution.
6. All modified artifacts pass markdownlint (P-008), YAML frontmatter validity, and cross-reference integrity checks.

## Blast Radius Assessment

Surfaces touched: policy registry template, one instructions file (install + harden), installed ship agent, ship agent template, pr-lifecycle skill template. No schema changes. No CLI changes. No distribution-layer changes. All changes are additive gate steps and a new policy entry.

**Requires plan hardening: no** — medium complexity, not elevated blast radius per P-006 criteria.
