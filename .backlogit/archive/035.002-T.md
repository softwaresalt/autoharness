The working-tree release.yml is a near-complete PyPI pipeline but is missing one critical hardening: the checkout step does not set `submodules: false`. When submodules are restored (T1), CI will attempt to recursively init them during the release job unless explicitly disabled — exactly the failure mode we are solving at the distribution layer.

Change required (single addition):
```yaml
- name: Checkout
  uses: actions/checkout@34e114876b0b11c390a56381ad16ebd13914f8d5 # v4
  with:
    submodules: false
```

Verify:
- `submodules: false` appears in the checkout step of .github/workflows/release.yml
- No other checkout steps in release.yml are missing this setting
- `test_release_workflow_publishes_to_pypi` still passes (this test does NOT assert the submodules setting, so no test change needed)

Commit message: `ci: disable submodule checkout in release workflow`

Width constraint: this task touches only .github/workflows/release.yml. Do not bundle other workflow changes here.
