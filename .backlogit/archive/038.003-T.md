## What

Add explicit graphtor-docs overlay weaving to both pipeline agents in the dogfood harness. graphtor-docs is a Primitive 1 overlay — it deepens State, Context, and Knowledge Retrieval by making indexed documentation the first-resort resolution path before file-based fallback.

### `stage.agent.md` additions

**Session start protocol** (after Engram check, Step 0.1c):
- Step 0.1d: If `graphtor-docs` capability pack is active, call `get_status` to verify server reachability and index freshness. If unreachable, log `GRAPHTOR_UNAVAILABLE` and continue with file-based tools. Do not halt.

**Before planning work** (Step 3, before impl-plan and alongside Engram search):
- Call `research_topic` or `search_local_docs` to resolve domain concepts, referenced APIs, or architecture questions from the indexed documentation before formulating the plan.
- For relationship/impact questions: prefer Engram's `unified_search` or `impact_analysis`.
- For documented concepts, API references, design docs: prefer graphtor-docs `research_topic`.

### `ship.agent.md` additions

**Session start protocol** (alongside Engram and intercom checks):
- Same `get_status` check, same `GRAPHTOR_UNAVAILABLE` log on failure.

**Before build work** (before harness / build-feature skill invocation):
- Call `search_local_docs` or `search_semantic` to resolve any documentation questions about the feature scope before beginning implementation.

### Coordination note

When both Engram and graphtor-docs are active:
- Use Engram for code relationships, impact analysis, and git history
- Use graphtor-docs for documentation lookup, API references, and concept research
- Document this routing rule in both agents under a "Multi-pack Knowledge Retrieval" note

### Reference

Both agents must include: "See `.github/instructions/graphtor-docs.instructions.md` for full search protocol, server lifecycle, fallback rules."

## Acceptance Criteria

- `stage.agent.md` has graphtor-docs `get_status` check at session start + pre-planning `research_topic` / `search_local_docs` step

- Both agents document the `GRAPHTOR_UNAVAILABLE` fallback path
- Both agents document the Engram/graphtor-docs routing split (code relationships vs. doc lookup)
