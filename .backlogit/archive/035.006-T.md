## Validation dry run

After all code tasks (T1–T5) are committed to a branch and CI is green:
1. Create a pre-release tag locally: `git tag v1.4.2-rc.1` (or the final version tag)
2. Push: `git push origin v1.4.2-rc.1`
3. Observe the release workflow in GitHub Actions — confirm each step passes through: Validate tag → Extract changelog → Build wheel → Twine check → Pre-publish check → PyPI publish → Smoke test → GitHub Release
4. If smoke test passes (confirms `uv tool run ... autoharness version` returns the version and templates/schemas exist), the pipeline is validated.
5. Delete the rc tag if using a pre-release: `git push origin :v1.4.2-rc.1`
6. Push the final tag: `git tag v1.4.2 && git push origin v1.4.2`
