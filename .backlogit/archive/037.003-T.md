## What

Add explicit agent-intercom overlay weaving to both pipeline agents in the dogfood harness, consistent with the overlay rules in `harness-architecture.instructions.md` and the installed `agent-intercom.instructions.md`.

### `stage.agent.md` additions

**Session start protocol** (after backlogit index sync, Step 0.1):
- Step 0.1c: If `agent-intercom` capability pack is active, call heartbeat/ping with a concise session-start status. If intercom is unreachable, log `INTERCOM_DEGRADED` and continue — do not halt.

**Before presenting choices to operator** (stash triage, plan review, shipment assembly):
- Broadcast self-contained summaries per the combined intercom+backlogit rule in `agent-intercom.instructions.md`: include item ID, priority, type, one-line summary, and recommended ordering. State that operator confirmation is awaited.

**Before destructive backlog operations** (archive, delete, status move to terminal states):
- Run intercom auto-check step. If not auto-approved, request operator clearance before proceeding.

### `ship.agent.md` additions

**Session start protocol** (after tool gate):
- Same heartbeat/ping check, same `INTERCOM_DEGRADED` log on failure.

**Phase broadcasts** (at planning started, task claimed, task completed, review complete, runtime verification, operational closure):
- Broadcast concise status per the Progress Protocol in `agent-intercom.instructions.md`.

**Destructive action gate** (file deletions, directory removals):
- Run intercom auto-check step before executing. Block if auto-check fails and intercom is unavailable.

### Reference

Both agents must include: "See `.github/instructions/agent-intercom.instructions.md` for full heartbeat, broadcast, approval, and degraded-mode rules."

## Acceptance Criteria

- `stage.agent.md` has intercom startup ping + broadcast-before-operator-choice step + destructive-action gate
- `ship.agent.md` has intercom startup ping + phase broadcast points + destructive-action gate
- Both agents document the `INTERCOM_DEGRADED` fallback path
- Cross-reference to `agent-intercom.instructions.md` present in both agents

## Constraint

Two agent files, ~45–60 min total. Width-isolated to agent definitions only.

## Dependency

IC-B must be complete first (instruction file must exist before agents cross-reference it).
