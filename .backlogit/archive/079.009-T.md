Add the epoch-level route, operational, and outcome roll-up fields needed for tool offload and expected-vs-actual gap reporting without creating a live event stream.

<!-- BEGIN:acceptance-criteria -->
- Tests are written first for route-kind serialization, operational offload counts, expected-vs-observed tool counts, scalar/map invariant totals, and distinct outcome gap/degraded/failure counts.
- `RouteConfiguration` exposes `route_kinds` and `primary_route_kind`; no other payload owns those fields.
- `OperationalReality` exposes epoch-level aggregate fields exactly as assigned by the decision matrix: `tool_surfaces`, `retrieval_packs`, `route_kind_counts`, `routed_lookup_count`, `raw_file_read_count`, `raw_search_count`, `avoided_file_read_count`, `tool_output_bytes`, `expected_tool_count`, `observed_expected_tool_count`, `missing_expected_tool_count`, `expected_tool_counts`, `observed_tool_counts`, `missing_expected_tool_counts`, `degraded_tool_count`, and `stale_or_unavailable_index_count`.
- `degraded_tool_count` is the routing/offload diagnostic count for expected or routed tools invoked through degraded fallback behavior; `tool_degraded_count` is the `AbsoluteOutcome` execution-status count for operations whose outcome was degraded. They are not aliases, have no equality invariant, and neither is summed into `tool_gap_count`.
- Expected-but-never-invoked tools are representable without any event row: `expected_tool_counts["engram.map_code"] = 1`, `observed_tool_counts["engram.map_code"] = 0`, and `missing_expected_tool_counts["engram.map_code"] = 1`.
- Tests assert scalar/map invariants: `expected_tool_count`, `observed_expected_tool_count`, and `missing_expected_tool_count` equal the sums of their maps, and `tool_gap_count = missing_expected_tool_count`. Tests also assert the per-tool derivation `missing_expected_tool_counts[tool] = max(expected_tool_counts[tool] - observed_tool_counts[tool], 0)`; `observed_tool_counts[tool]` counts observed invocations of an expected tool, so over-observation clamps missing to `0`.
- Populated operational and `AbsoluteOutcome` metric/count fields added by this task must have same-named entries in both `metric_sources` and `metric_quality`, including `tool_failure_count`, `tool_degraded_count`, and `tool_gap_count`, matching the epoch provenance invariant from 079.001-T.
- `RouteConfiguration`/`AbsoluteOutcome` roll-ups remain safe summaries; no raw tool output, raw argv, live event payload, or CozoDB/agent-engram ingestion code is added.
<!-- END:acceptance-criteria -->
