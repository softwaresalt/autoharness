## Cross-Reference vs. autoharness Current Personas

**autoharness review/SKILL.md.tmpl currently has:**

Always-on:
- Constitution Reviewer (autoharness-specific)
- `{{PRIMARY_LANGUAGE}}` Reviewer (technology template)
- Learnings Researcher

Conditional:
- Architecture Strategist (module boundaries, new abstractions)
- Concurrency Reviewer (async/concurrent patterns)
- Scope Boundary Auditor (multi-domain changes)
- Agent-Native Parity Reviewer (MCP/agent surfaces)
- Security Reviewer (auth, endpoints, input handling)

**autoharness review template agent files:**
`constitution-reviewer`, `technology-reviewer`, `architecture-strategist`, `concurrency-reviewer`, `scope-boundary-auditor`, `agent-native-parity-reviewer`, `security-reviewer`, `security-lens-reviewer`, `language-engineer`

**Gap summary**: autoharness is missing correctness, maintainability, adversarial, performance, reliability, data-migrations, and api-contract reviewers.

## Recommendations

### Priority 1: Promote to Always-On

| Persona | Rationale |
|---|---|
| **Correctness Reviewer** | Logic errors and edge cases are the most universal failure mode in any codebase. ATV marks this as always-on. autoharness's primary language reviewer covers language safety but not behavioral correctness. This fills a genuine gap. |
| **Maintainability Reviewer** | Premature abstraction, dead code, and coupling are universal across all languages and template types. Aligns with Simplicity First (karpathy) and Principle VI (autoharness constitution). |

### Priority 2: Add as Conditional

| Persona | Activation Signal | Rationale |
|---|---|---|
| **Adversarial Reviewer** | diff ≥ 50 lines OR diff touches high-risk domains | Uniquely valuable for catching emergent failures between components. ATV has a calibrated depth system (Quick/Standard/Deep) tied to diff size and risk signals. Domain-agnostic. |
| **Performance Reviewer** | diff touches db queries, loops over large collections, caching, I/O-bound paths | Domain-agnostic pattern (N+1, unbounded memory growth, hot-path allocations). High confidence threshold prevents false positives on cold paths. |
| **Reliability Reviewer** | diff touches error handling, retries, circuit breakers, timeouts, async handlers | Directly aligned with Primitive 5 (Guardrails). Complements autoharness's circuit-breaker and concurrency instruction templates. |
| **Data Migrations Reviewer** | diff touches migration files, schema changes, data transforms | Relevant for any workspace with a data store. Migration safety (rollback plan, backwards compatibility, NULL constraints) is universally applicable. |
| **API Contract Reviewer** | diff touches API routes, request/response types, serialization, exported type signatures | Universal for any workspace exposing an API. Breaking contract changes are a high-blast-radius failure mode. |

### Defer

- `code-simplicity-reviewer`: Overlaps with Maintainability Reviewer and karpathy-guidelines (if adopted from 011.001). Defer unless a "final simplicity pass" use case emerges.
- `design-implementation-reviewer`: Niche — only valuable in design-heavy workflows with formal design docs. Add as an optional capability pack overlay.
- `previous-comments-reviewer`: Better handled by the fix-ci skill's PR comment thread logic. Not a general code reviewer.
- `project-standards-reviewer`: Requires compound-engineering plugin. Not portable.
- Rails/language-specific reviewers: Already handled by `{{PRIMARY_LANGUAGE}}` reviewer template.
- Document review personas (coherence, feasibility, design-lens, product-lens): Scope is plan review and document review — different skill, not code review.

## Proposed Activation Signals

```text
# Correctness Reviewer — always-on (no activation condition)

# Maintainability Reviewer — always-on (no activation condition)

# Adversarial Reviewer
diff_size >= 50 OR diff_touches_patterns:
  - auth*, authorization*, permission*, credential*, session*
  - payment*, billing*, migration*, backfill*
  - external_api*, webhook*

# Performance Reviewer
diff_touches_patterns:
  - *.sql, *query*, *queries*, *cursor*
  - loop*, iterate*, range*, batch*
  - cache*, memoize*, buffer*
  - file_read*, stream*, download*

# Reliability Reviewer
diff_touches_patterns:
  - *error*, *retry*, *timeout*, *circuit*
  - async*, goroutine*, thread*, concurrent*
  - job*, worker*, queue*, background*

# Data Migrations Reviewer
diff_touches_patterns:
  - */migrations/*, *_migration.*, *migration_*
  - *schema*, *migrate*, *backfill*

# API Contract Reviewer
diff_touches_patterns:
  - */routes/*, */handlers/*, */controllers/*
  - *_api.*, *_endpoint.*, *serializer*
  - */proto/*, *.proto
  - exported type signature changes (language-specific)
```
