# Spike Findings: plugingen Audit Pattern for Portability Verification

## ATV Audit Rule Taxonomy

`audit.go` implements a deterministic, sorted audit of harness artifacts against three rule categories:

### Rule Category 1: Plugin Cache Path References

```go
label: "compound-engineering plugin cache path"
re: `(?i)(?:~/\.copilot/plugins/cache/[^"\s]*compound-engineering|plugins/compound-engineering/)`
```

**What it catches**: Hardcoded paths to a specific plugin's cache directory in the user home. Users who install via marketplace won't have this path. Skills/agents referencing it will silently fail.

**Why it matters**: Path references tied to a specific installation layout break portability across users, environments, and install methods.

### Rule Category 2: Namespaced Plugin Tool References

```go
label: "compound-engineering namespaced agent or tool"
re: `compound-engineering[:_][a-z][a-z0-9-]*(?::[a-z0-9*-]+)?`
```

**What it catches**: References to agents/tools in the `compound-engineering` namespace (e.g., `compound-engineering:research:learnings-researcher`). These references only work when that specific plugin is installed alongside the skill.

**Why it matters**: A harness artifact referencing a specific plugin's namespace creates an undeclared dependency that breaks silently for users who don't have that plugin.

### Rule Category 3: MCP Plugin Tool Name Format

```go
label: "MCP plugin tool name (mcp__plugin_...)"
re: `mcp__plugin_[a-zA-Z0-9_-]+__[a-zA-Z0-9_-]+`
```

**What it catches**: References like `mcp__plugin_backlogit__sync_index` — the runtime-resolved MCP tool name format for Copilot CLI plugins. This format is Copilot CLI-specific and breaks in other environments (Cursor, Claude Code, Codex, VS Code direct MCP).

**Why it matters**: A harness artifact hardcoding a Copilot CLI MCP plugin name format is IDE-specific — it violates environment agnosticism.

### The Allow-List Pattern

ATV's audit allow-lists known dependencies with **documented reasons** and **graceful degradation expectations**. Additions to the allow-list require explicit justification. This "document the dependency, enforce the exception" pattern is the key design principle.

### Generate() as Gate

`generate.go` runs `Audit()` as the first step of `Generate()`. If the audit returns findings, `Generate()` returns an error and nothing is written. This makes portability a **mandatory gate**, not an advisory check.

## Candidate Portability Rules for autoharness

Mapping ATV rule categories to autoharness-specific violations:

### Category A: Hardcoded Host-Specific Paths

| Pattern | Regex | What it catches |
|---|---|---|
| Unix home dir references | `~/\.[a-z]` | `~/.autoharness/`, `~/.copilot/`, `~/.config/` in generated artifacts |
| Windows home dir references | `%USERPROFILE%`, `%APPDATA%`, `C:\\Users\\` | Windows-specific absolute paths in cross-platform templates |
| Hardcoded autoharness home | `/home/[^/]+/\.autoharness`, `~/.autoharness` | Hardcoded autoharness home path (should use `AUTOHARNESS_HOME` env var or runtime resolution) |

**autoharness violation example**: A skill template that says "run `~/.autoharness/bin/autoharness`" instead of "run `autoharness`". The installed artifact would hardcode the prior user's home path.

**Expected status**: Currently not checked. Workspace profile uses `workspace_path: "."` convention (from dogfood memory), but no assertion enforces that generated artifacts don't contain hardcoded paths.

### Category B: Environment-Specific Agent/Skill Directory References

| Pattern | Regex | What it catches |
|---|---|---|
| Local-agents directory reference | `\.github/local-agents` | `.github/local-agents/` references in installed harness artifacts — VS Code + Copilot CLI concept; other environments (Cursor, Claude Code, Codex) don't use it |
| VS Code settings references | `\.vscode/settings\.json` | Settings path references in installed artifacts — IDE-specific |
| IDE-specific config paths | `\.claude/\|\.cursor/` | Tool-specific config directory references in environment-agnostic installed artifacts |

**autoharness violation example**: A generated harness agent that says "add the agent to `.github/local-agents/`" — this path doesn't exist in Cursor or Claude Code environments. Start scripts legitimately use this path, but harness *agent/skill/instruction* files must not reference it.

**Note**: `start.ps1` and `start.sh` are legitimately environment-specific scripts and should be in an allow-list.

### Category C: Hardcoded MCP Tool Names

| Pattern | Regex | What it catches |
|---|---|---|
| Copilot CLI MCP plugin format | `mcp__plugin_[a-zA-Z0-9_-]+__[a-zA-Z0-9_-]+` | Direct use of the Copilot CLI MCP plugin name format |

**autoharness violation example**: An installed skill file that calls `mcp__plugin_backlogit__sync_index` directly. This format only works in Copilot CLI; other environments (Cursor, Claude Code, Codex) use different tool name conventions.

**Expected status**: Installed harness artifacts should never contain raw `mcp__plugin_...` strings — template variables (`{{OP_*}}`) are resolved at install time. `verify_workspace.py` does not currently scan installed artifacts for this pattern. This portability assertion would catch cases where variable substitution failed or was bypassed.

> **Separate concern — template-lint**: `.tmpl` source files should also be checked that every MCP tool call uses a `{{OP_*}}` placeholder rather than a hardcoded name. This is a *template authoring lint*, not a portability scan of installed artifacts, and belongs in a distinct `TEMPLATE_LINT_ASSERTIONS` group. The portability scan (installed artifacts only) and the template lint (`.tmpl` sources only) address different failure modes and must be implemented separately.

## Recommendation: Adopt

Add a `PORTABILITY_ASSERTIONS` group to `verify_workspace.py` that scans installed harness artifacts (`.github/agents/`, `.github/skills/`, `.github/instructions/`, `.github/prompts/`) for portability violations.

**Design decisions informed by ATV**:
1. **Sorted, deterministic findings**: iterate artifact files in sorted order so findings are reproducible
2. **Allow-list with documented reasons**: known acceptable violations (e.g., start scripts) are in a structured allow-list, not ignored silently
3. **Gate behavior**: portability assertions should be run as part of the verify step (post-install) and the tune step (drift detection)
4. **Line-number-aware**: findings include file and line number for actionable remediation

### Proposed verify_workspace.py Assertion Structure

```python
PORTABILITY_RULES = [
    {
        "label": "hardcoded user home path",
        "pattern": r"~/\.[a-zA-Z]|%USERPROFILE%|%APPDATA%|C:\\\\Users\\\\",
        "severity": "P1",
    },
    {
        "label": "local-agents directory reference in harness artifact",
        "pattern": r"\.github/local-agents",
        "severity": "P1",
    },
    {
        "label": "MCP plugin tool name format (environment-specific)",
        "pattern": r"mcp__plugin_[a-zA-Z0-9_\-]+__[a-zA-Z0-9_\-]+",
        "severity": "P1",
    },
    {
        "label": "hardcoded autoharness home path",
        "pattern": r"~/\.autoharness|~\\\.autoharness",
        "severity": "P1",
    },
]

# Allow-list contract: each entry exempts a (rule_label, file_glob) pair.
# - `rule` must match a `label` in PORTABILITY_RULES exactly.
# - `file_glob` is matched against the artifact's workspace-relative path using fnmatch.
# - This makes exemptions deterministic: a finding is suppressed only when BOTH
#   the rule label AND the file path match. Exemptions are never content-pattern-based.
PORTABILITY_ALLOW_LIST = [
    {
        "rule": "local-agents directory reference in harness artifact",
        "file_glob": "start.*",
        "reason": "start scripts legitimately reference .github/local-agents for VS Code+Copilot-CLI setup",
    },
    # future additions require both a matching rule label and a specific file_glob
]

PORTABILITY_ASSERTIONS = [
    {
        "id": "portability_scan",
        "description": "Installed harness artifacts must not contain environment-specific paths or tool names",
        "scan_dirs": [".github/agents/", ".github/skills/", ".github/instructions/", ".github/prompts/"],
        "rules": PORTABILITY_RULES,
        "allow_list": PORTABILITY_ALLOW_LIST,
    }
]
```

**Scan scope**: Installed harness artifacts only (`.github/{agents,skills,instructions,prompts}/`). Template source files (`.tmpl`) are explicitly exempt: they legitimately contain `{{OP_*}}` placeholders that resolve to environment-specific MCP tool names at install time. Checking `.tmpl` sources for hardcoded MCP names is a separate template-lint concern (see Category C note above).

**Not scanned**: `start.ps1`, `start.sh`, `.vscode/settings.json` — these are legitimately environment-specific.
