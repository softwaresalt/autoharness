# Shipment Integrity Gates — Implementation Plan

## Problem Frame

The `backlogit_ship_shipment` MCP tool does not validate per-item completion
state before archiving. It will archive items that are still `queued`, `active`,
or missing from disk. Additionally, a known tool quirk deletes archive copies
from the working tree after moving them. These gaps were discovered in
production use in the agent-engram workspace, where:

1. Stage over-included pre-existing queue items into a shipment manifest
2. Ship archived incomplete items without detecting the mismatch
3. Archive files were silently deleted from the working tree after archival

The agent-engram workspace developed compensating controls (shipment-reconcile
skill, scope guard, P-007 policy). These are proven and need to be upstreamed
into autoharness templates so every workspace with `{{FEATURE_SHIPMENTS}}`
inherits them.

## Decisions and Rationale

| Decision | Rationale |
|---|---|
| Extract from agent-engram rather than writing from scratch | The skill is proven in production use; templatization preserves the proven behavior |
| Gate everything on `{{FEATURE_SHIPMENTS}}` | Workspaces without shipment support should not see these artifacts |
| Use `{{OP_*}}` variables, not hardcoded tool names | Maintains tool-agnostic design; works with future backlog tools that support shipments |
| P-007 as a separate policy, not inline in ship agent | Policies are the system of record for gates; the ship agent references the policy |
| Scope guard in Stage, not Ship | Stage is where scope decisions are made; Ship should receive a clean manifest |

## Risks and Caveats

| Risk | Mitigation |
|---|---|
| Step renumbering in ship.agent.md.tmpl could break cross-references | Verify all step references in the file after insertion |
| Agent-engram version has hardcoded lock file path `.backlogit/queue/{shipment_id}.S.md` | Templatize as `{{BACKLOG_DIRECTORY}}/queue/{shipment_id}.md` |
| The `git restore` workaround is specific to a backlogit tool quirk | P-007 documents this explicitly; if backlogit fixes the quirk upstream, the policy becomes a no-op guard rather than actively harmful |
